Overview

Debt Management System is a production-style mobile application built for retail shop owners to manage customer debts, credits, and outstanding balances efficiently across multiple devices.

The application provides secure account-based access where each shop owner can manage only their own customer records. It supports real-time cloud synchronization using a Node.js REST API and MongoDB Atlas while maintaining encrypted local storage for improved security and offline persistence.

The Problem

"Managing customer debts manually in notebooks caused calculation errors, missing payments, and difficulty tracking balances."

— Retail Shop Requirement

Traditional debt tracking methods used by small businesses lacked security, searchability, automatic calculations, and reliable backup systems. Shop owners often struggled to track credits, debt settlements, and remaining balances accurately.

Architecture

The application follows a modern client-server architecture with Flutter as the mobile frontend and a REST API backend built using Node.js and Express.js.

Key Engineering Decisions

01

Riverpod for scalable state management

Riverpod provided predictable state handling, improved code organization, and simplified reactive UI updates across the application.

02

JWT-based authentication

Implemented secure token-based authentication with account isolation so each user can access only their own customer and transaction records.

03

AES-256 encrypted local storage

Sensitive user data and cached records are encrypted locally to improve security and protect business information.

04

Automatic balance calculation system

The system dynamically calculates outstanding balances by subtracting credits from total debts, reducing manual calculation errors.

Lessons Learned

Building this project strengthened my understanding of full-stack mobile application architecture, authentication systems, API development, and cloud database deployment.

One major learning was designing secure multi-user data isolation to ensure that each account could only access its own records. I also gained practical experience handling encrypted local storage, REST API integration, and production deployment challenges on free-tier cloud infrastructure.
